🔐 Secure IoT Access and Threat Monitoring System
A full-stack cybersecurity platform for IoT device authentication, real-time threat detection, automated incident response, and AI-powered security analysis.

📌 Overview
This system provides secure mutual authentication for IoT devices using HMAC-SHA256 challenge-response, monitors security events through a custom SIEM engine, automates threat responses via a SOAR pipeline, maintains a tamper-evident blockchain audit trail, and now features an AI-powered SOC Analysis engine (via Ollama + Llama3) — all managed through a React admin dashboard.

🤖 AI SOC Analysis (New — In Progress)
A new AI Analysis page integrates a locally-running Ollama instance (Llama3 model) to act as an AI Security Operations Center analyst.

How it works
Admin selects filters (date range or single date, target device)
Admin selects a report type
The system fetches filtered security events + alerts from the database
Events are passed as context to the Ollama Llama3 model via its local API
The AI generates a structured security report and streams it to the dashboard
Report Types
Report Type	Description
Risk Summary	Overall risk score, threat classification, and executive summary
Full SOC Report	Deep-dive: incident timeline, event correlations, blast radius, impact assessment
Alert Analysis	Attack pattern breakdown, severity reasoning, and prioritized mitigation steps
Filter Options
Date Filter — Single date or date range
Device Filter — All Devices, or a specific registered device (ESP32_01, esp32_01, etc.)
Requirements
To use the AI Analysis page, you need Ollama installed and running locally:

# Install Ollama (https://ollama.com)
ollama pull llama3
ollama serve
Ollama runs on http://localhost:11434 by default. The backend calls this endpoint when generating reports.

Note: AI Analysis works fully offline — no OpenAI API key or internet connection required. Everything runs on your local machine.

from app import create_app
from models import db, BlockedIP, Device
app = create_app()
with app.app_context():
    BlockedIP.query.delete()
    for d in Device.query.filter_by(status='LOCKED').all():
        d.status = 'ACTIVE'
    db.session.commit()
    print('Reset complete')
"

🛡 Security Hardening
Role escalation blocked on /register — public users can only get viewer or user
HMAC validated as 64-char lowercase hex before any DB lookup
CORS restricted to http://localhost:8080 only via .env
debug=False by default — controlled via FLASK_DEBUG env variable
JWT expiry enforced via JWT_EXPIRES_MINUTES env variable
Input length limits on all endpoints (username: 50, password: 128, device_id: 100)
db.session.rollback() in every exception handler
Nonce deleted before HMAC comparison — closes replay window
hmac.compare_digest() for constant-time comparison
Same error message for wrong username vs wrong password — prevents user enumeration
Pagination enforced on all list endpoints — no unbounded .query.all()
